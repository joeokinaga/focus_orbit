## B. セッション状態機械(§4 の実機写像)

- [x] **B1 完走**: 1分プリセットで開始 → 60秒静置 → completed 画面(報酬 **100** 表示)
- [x] **B2 warning→復帰**: 開始後、机を軽く叩いて微振動 → warning 表示(点滅+猶予カウント)→ 静置 → running に復帰する
- [x] **B3 graceTimeout**: warning を誘発し、そのまま10秒間刺激を続ける → aborted(猶予切れ)→ acknowledge → idle
- [x] **B4 lifted**: running 中に端末を持ち上げる → 即 aborted(持ち上げ)
- [x] **B5 systemInterrupted**: running 中にホーム画面へ出る → アプリに戻る → aborted(システム中断)になっている(D3: バックグラウンド=中断)
- [x] **B6 userCancelled**: running 中に中断ボタン → 確認ダイアログ → 「中断する」→ aborted。「続ける」なら running 継続
- [x] **B7 ViewMode 直交性(D2)**: running/warning 中のみ orbit 表示へ切替可能。idle では切替不可

> **B2〜B4 が反応しない場合**: 不合格ではなく **LANDMINE ③(端末別キャリブレーション)** の発動。`flutter run -t lib/main_sensor_debug.dart` で生値と閾値を観測し、gravity-z の符号と振幅レンジを記録して報告すること。閾値調整は別タスクとして切る。
